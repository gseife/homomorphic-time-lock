Slide 4

Motivation: The Problem

  • E-voting without trusted authority (blockchain setting)
  • Commit-reveal fails if voters go offline (bias attack)
  • Standard TLPs fix this — but don't scale (n puzzles to solve)

Slides 5–7

Concept 1
Homomorphic Encryption

  • Core idea: compute on ciphertexts without decrypting
  • Analogy: sealed ballot box that outputs only the total
  • Three flavors: additive, multiplicative, fully homomorphic
  • Paper focuses on additive (voting) & multiplicative (contracts)

Slides 8–11

Concept 2
Time-Lock Puzzles

  • Lock a secret for time T using repeated squaring mod N
  • Creator is fast (knows factorization), solver is slow (T steps)
  • Inherently sequential — parallelism doesn't help
  • Scalability problem: n missing votes = n × T work

Slides 12–15

Core Contribution
Homomorphic Time-Lock Puzzles

  • 4 algorithms: PSetup, PGen, PEval (new!), PSolve
  • LHTLP construction: Paillier-style encoding in Z*N²
  • Multiplying puzzles = adding secrets (with visual example)
  • Security: Sequential Squaring + DDH + DCR assumptions

Slides 16–17

Application 1
Multi-Party Coin Flipping

  • n parties each lock a random bit, broadcast puzzles
  • Combine all → solve 1 puzzle → take last bit = fair coin
  • Complexity: n × T (old) vs 1 × T (HTLP)
  • Relevant for blockchain randomness (e.g., Ethereum)

Slides 18–20

Application 2
E-Voting

  • Each vote = vector of m puzzles (1 for chosen, 0 for rest)
  • Combine per-candidate, solve m puzzles (not n!)
  • Security: ballot secrecy, no bias, offline-resilient
  • No trusted tallying authority — fully public counting

Slides 21–23

  • Voter page (phone) + Admin dashboard (presenter screen)
  • 128-bit primes, ~150 sec solving time per candidate
  • Code walkthrough: PGen, PEval, PSolve, vote casting
